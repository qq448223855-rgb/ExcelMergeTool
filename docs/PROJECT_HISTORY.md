# 项目历史

本文档整理了 Excel 合并工具从最初代码到 V1.0.1 的主要需求和决定，供后续
对话与开发直接使用。

## 项目定位

- 面向非技术用户的 macOS Excel 批量合并工具。
- 当前正式支持 Apple 芯片 Mac。
- 输入支持 `.xlsx` 和 `.xlsm`，输出为 `.xlsx`。
- 使用 PySide6 构建桌面界面，使用 openpyxl 处理工作簿。

## V1.0.0 已完成

- 添加文件时可正常进入文件夹选择文件。
- 添加文件夹后在界面表格中显示文件名、大小、行数和路径。
- 禁用列表拖拽，改用“上移 / 下移”按钮，避免拖动导致文件被删除。
- 后续文件可选择跳过 `0-99` 行。
- 删除“保留行高”选项。
- 增加保存路径选择及路径显示，未选择保存位置时不能开始合并。
- 保存和覆盖窗口跟随电脑系统语言。
- 修复公式生成后被 Excel 报告损坏的问题。
- 版本从早期测试命名统一为 `V1.0.0`。
- 首次源码和 macOS 应用已发布到 GitHub。

## V1.0.1 目标

- 不改变用户操作方式。
- 普通文件自动使用流式读取和写入。
- 存在需要保留的合并单元格时自动使用兼容模式。
- 用 ZIP/XML 元数据扫描替代文件信息阶段的完整 openpyxl 扫描。
- 缓存重复样式，降低大表处理时间和内存占用。
- 精简不使用的 Qt 网络、TLS、SVG、图片插件和无关翻译资源。
- 保留全部 `qtbase` 系统语言翻译。

## 性能基线

测试目录：

`1781228553600.0.VatapplExcel.xlsx`

包含 4 个文件，总计 70,453 行、37 列。V1.0.0 合并耗时约 32.6 秒；
V1.0.1 的验收目标是不超过 27 秒。

## 版本和发布规则

- 版本名称使用 `Excel合并工具V主版本.次版本.修订号`。
- 新版本不覆盖旧版本。
- Git 标签使用小写前缀，例如 `v1.0.1`。
- macOS 发布包命名为
  `Excel合并工具V1.0.1-macOS-arm64.zip`。
